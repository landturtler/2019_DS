데이타 베이스를 만들고자 한다. 
데이타 베이스는 학생 N명에 대한 신상 기록을 저장하고 있다. 
1. 각각의 레코드는 아래와 같이 출석번호, 이름, 성적(0~100) 으로 구성된다. 
struct student { 
int num; 
char name[10]; 
int score; 
} 
2. DB는 파일에 저장되어 있고, 검색/삽입/삭제/소팅 등의 동작을 수행할 수있다. 
인터페이스 명령어는 
1. insert num name height score : 번호의 위치에 해당 레코드를 삽입 
2. delete name : 입력 이름에 해당하는 레코드를 삭제 
3. list name : 입력 이름에 해당하는 레코드의 기록을 보여줌 
4. sort name : 이름순으로 정렬 
5. sort score : 성적순으로 정렬 
6. read DB이름 : 해당 파일을 read목적으로 open 하고 data를 메모리로 읽음 
7. write DB이름 : 해당 파일을 write목적으로 open 하고 파일에 내용을 적음. 
3. 파일에 몇명의 DB가 있는지는 그 파일을 읽어서 확인하고, 그에 맞는 크기만큼의 메모리를 
동적으로 할당함. 
4. 배열의 방법으로 DB 처리하는 방법과, Linked list로 DB 처리하는 방법 모두를 구현할 것 
단 인터페이스는 동일하게 할 것 (즉, command는 동일하게) 
5. 처음에는 random하게 만명의 데이터를 발생시켜서 파일에 적고, 
DB command를 사용하여 DB를 변화시킬 것.. 
6. DB의 크기를 늘리면서 크기 N에 따른 정렬(sorting) 에 소요되는 시간을 graph로 
plot 해 볼것. 
7. data를 insert/delete 할 때, 배열로 한 경우와 리스트로 처리한 경우의 시간을 비교해 볼것 
8. 데이터 검색에 소요되는 시간을 줄이는 여러 가지 방법을 생각해 보고, 코딩해 볼 것...
